Documentation du Gestionnaire de Tâches

Système de gestion de tâches en C++ utilisant la bibliothèque nlohmann/json pour la persistance des données.

1. Introduction

1.1 Description générale

Ce projet implémente un système complet de gestion de tâches (TODO List) en C++. Il permet d'effectuer toutes les opérations CRUD (Create, Read, Update, Delete) sur des tâches stockées de manière persistante dans un fichier JSON. Le programme fonctionne en ligne de commande et utilise des codes couleurs ANSI pour améliorer la lisibilité des sorties.

1.2 Caractéristiques principales

Fonctionnalité Description
Création de tâches Ajout de nouvelles tâches avec génération automatique d'ID
Lecture des tâches Affichage formaté de toutes les tâches
Mise à jour Modification du titre des tâches existantes
Suppression Retrait de tâches par identifiant
Gestion du statut Basculement du statut de complétion
Recherche Consultation détaillée d'une tâche spécifique

1.3 Technologies utilisées

  • Langage : C++ (Standard C++11 minimum)
  • Bibliothèque JSON : nlohmann/json (v3.x)
  • Stockage : Fichier JSON (output.json)
  • Interface : Ligne de commande (CLI)

2. Installation

2.1 Prérequis système

Composant Requis Notes
Compilateur C++ g++ 4.8+ ou clang++ 3.3+ Support C++11 minimum
nlohmann/json Version 3.0+ Bibliothèque header-only
Système d'exploitation Linux, macOS, Windows Terminal avec support ANSI

2.2 Installation de nlohmann/json

Ubuntu/Debian :

sudo apt-get update
sudo apt-get install nlohmann-json3-dev

macOS :

brew install nlohmann-json

Installation manuelle :

# Télécharger json.hpp depuis :
# https://github.com/nlohmann/json/releases
# Placer le fichier dans le répertoire du projet

4. Utilisation générale

4.1 Syntaxe générale

./todo <commande> [arguments]

4.2 Liste des commandes

Commande Arguments Description
create <titre> Crée une nouvelle tâche
read - Affiche toutes les tâches
update <id> <titre> Modifie une tâche existante
delete <id> Supprime une tâche
check <id> Bascule le statut de complétion
search <id> Recherche une tâche par ID

5. Commande create

5.1 Description

La commande create permet de créer une nouvelle tâche dans le système. L'identifiant est généré automatiquement en incrémentant le dernier ID existant. Par défaut, la tâche est créée avec le statut "non complétée".

5.2 Syntaxe

./todo create <titre>

5.3 Paramètres

Paramètre Type Obligatoire Description
titre string Oui Le titre de la tâche à créer

5.4 Exemple d'utilisation

$ ./todo create "Acheter du pain"

✓ Task created successfully!
  ID: 1
  Title: Acheter du pain

5.5 Valeur de retour

Code Signification
0 Succès - Tâche créée avec succès
1 Erreur - Arguments manquants

5.6 Fonction interne associée

void Create(const Item &task, json &j)

Cette fonction ajoute une nouvelle entrée au tableau JSON et incrémente le compteur global de tâches.

6. Commande read

6.1 Description

La commande read affiche l'intégralité du contenu du fichier JSON contenant toutes les tâches. L'affichage est formaté avec une indentation de 4 espaces pour une meilleure lisibilité.

6.2 Syntaxe

./todo read

6.3 Paramètres

Cette commande ne requiert aucun paramètre.

6.4 Exemple d'utilisation

$ ./todo read

═══════════════════════════
    JSON File Content
═══════════════════════════

[
    {
        "Id": 1,
        "Title": "Acheter du pain",
        "Completed": false
    }
]

6.5 Fonction interne associée

void Read(json &j)

Cette fonction utilise la méthode dump(4) pour afficher le contenu JSON avec indentation.
